
- Local repositories are stored on your computer.
-  Remote repositories are hosted online such as on GitHub.
- A repository is a project folder tracked by Git.
-  A commit saves changes.
-   A branch allows separate development without affecting the main project.
- The staging area is where files are prepared before committing changes.
- Push uploads local commits to GitHub.
- Push command also is the command that makes everything the same on each webiste. Make sure to ctrl S and save your work on VS code or it will not transfer over.
-  While pull downloads changes from the remote repository.
- A .gitignore file tells Git which files or folders should not be tracked.
- A README explains the purpose of the project and gives useful information for users or developers.
